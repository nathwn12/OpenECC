## Security Checks (Block)
- **SQL Injection**: String concatenation in database/sql queries
  ```go
  // Bad
  db.Query("SELECT * FROM users WHERE id = " + userID)
  // Good
  db.Query("SELECT * FROM users WHERE id = $1", userID)
  ```

- **Command Injection**: Unvalidated input in os/exec
  ```go
  // Bad
  exec.Command("sh", "-c", "echo " + userInput)
  // Good
  exec.Command("echo", userInput)
  ```

- **Path Traversal**: User-controlled file paths
  ```go
  // Bad
  os.ReadFile(filepath.Join(baseDir, userPath))
  // Good
  cleanPath := filepath.Clean(userPath)
  if strings.HasPrefix(cleanPath, "..") { return ErrInvalidPath }
  ```

- Race Conditions, unsafe package without justification, hardcoded secrets, insecure TLS, weak crypto

## Error Handling (Block)
- **Ignored Errors**: Using `_` to ignore errors
  ```go
  // Bad
  result, _ := doSomething()
  // Good
  result, err := doSomething()
  if err != nil { return fmt.Errorf("do something: %w", err) }
  ```

- **Missing Error Wrapping**: Errors without context
  ```go
  // Bad
  return err
  // Good
  return fmt.Errorf("load config %s: %w", path, err)
  ```

- Panic for recoverable errors, not using errors.Is/As for error checking

## Concurrency (Warn)
- **Goroutine Leaks**: Goroutines that never terminate
  ```go
  // Bad: No way to stop goroutine
  go func() { for { doWork() } }()
  // Good: Context for cancellation
  go func() {
    for { select { case <-ctx.Done(): return; default: doWork() } }
  }()
  ```

- Race conditions (run `go build -race ./...`)
- Missing sync.WaitGroup for coordination
- Mutex misuse — always `defer mu.Unlock()`
  ```go
  // Bad: Unlock might not be called on panic
  mu.Lock(); doSomething(); mu.Unlock()
  // Good
  mu.Lock(); defer mu.Unlock(); doSomething()
  ```

- Unbuffered channel deadlock, context not propagated

## Code Quality (Warn)
- Large functions (>50 lines), deep nesting (>4 levels)
- Interface pollution, package-level mutable state, naked returns
- Non-idiomatic code patterns

## Performance (Suggest)
- String building with `+=` in loops (use `strings.Builder`)
- Slice pre-allocation (`make([]T, 0, cap)`)
- N+1 queries in loops, missing connection pooling
- Inconsistent pointer vs value receivers

## Best Practices (Suggest)
- Accept interfaces, return structs
- Context should be first parameter: `func Process(ctx context.Context, id string)`
- Table-driven tests, godoc comments on exported functions
- Error messages lowercase, no punctuation

## Go-Specific Anti-Patterns
- **init() Abuse**: Complex logic in init functions
- **Empty Interface Overuse**: Using interface{} instead of generics
- **Type Assertions Without ok**: Can panic — always use `v, ok := x.(string)`
- **Deferred Call in Loop**: Resource accumulation — wrap in closure
